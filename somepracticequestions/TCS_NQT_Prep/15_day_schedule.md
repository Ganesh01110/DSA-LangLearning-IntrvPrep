# 15-Day Tight Preparation Schedule

Focus: 2 hours of coding practice + 1 hour of concept review daily.

| Day | Topic | Tasks |
| :--- | :--- | :--- |
| **1-3** | **Basic Math & Numbers** | Prime, Palindrome, GCD, LCM, Factorial, Series (AP/GP/Fibonacci). |
| **4-6** | **Arrays (Foundation)** | Searching, Sorting (Bubble/Selection), Second Max, Frequency, Duplicates. |
| **7-8** | **String Handling** | Vowels, Anagrams, Reversing, Toggle Case, Regex-like manual parsing. |
| **9-10** | **Advanced Arrays** | Kadane's, Sliding Window, Matrix (Spiral/Rotation), Subsets. |
| **11-12** | **Basic DSA** | Stack (Parenthesis), Queue basics, Linked List (Traverse/Reverse). |
| **13** | **Recursion & Greedy** | Simple recursion problems, Fractional Knapsack, Sorting by custom logic. |
| **14** | **Mock Test 1** | Solve 2 previous year NQT questions in 90 minutes. |
| **15** | **Mock Test 2 & Revision** | Identify weak areas (e.g., Python TLE or C++ STL) and refine. |

## Tight Preparation Rules:
1.  **No IDE Help:** Practice on a simple text editor or online compiler without auto-complete.
2.  **Manual I/O:** Always write your own input reading logic; don't rely on `Scanner` shortcuts.
3.  **One Language:** Master **one** language's syntax perfectly. Don't switch between C++ and Python mid-exam.
4.  **Timer:** Always set a 1-hour timer for each difficult problem.
